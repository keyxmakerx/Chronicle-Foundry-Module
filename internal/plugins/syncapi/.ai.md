# Sync API Plugin

## Purpose
Provides a secure REST API for external tool integration. External clients
(Foundry VTT, custom scripts, etc.) authenticate with API keys and can
read/write campaign data through versioned endpoints.

## Architecture
Standard plugin: handler ‚Üí service ‚Üí repository ‚Üí MariaDB.
Additionally provides API authentication middleware for the `/api/v1` routes.

### Files
- `model.go` ‚Äî Domain types: APIKey, APIRequestLog, SecurityEvent, IPBlock, stats.
- `repository.go` ‚Äî SQL operations for all sync API tables.
- `service.go` ‚Äî Business logic: key generation (bcrypt), auth, rate limiting, logging.
- `handler.go` ‚Äî HTTP handlers for management UI (admin dashboard + owner key pages).
- `api_handler.go` ‚Äî REST API v1 endpoint handlers (read, write, sync).
- `middleware.go` ‚Äî API key auth, permission checks, campaign match, rate limiting.
- `routes.go` ‚Äî Route registration: admin, campaign, and /api/v1 endpoints.
- `owner_keys.templ` ‚Äî Campaign owner key management page.
- `admin_dashboard.templ` ‚Äî Admin API monitoring dashboard.

### Database Tables
- `api_keys` ‚Äî API key registry (bcrypt-hashed keys, per-campaign, per-user).
- `api_request_log` ‚Äî Request audit trail (method, path, status, IP, latency).
- `api_security_events` ‚Äî Security events (rate_limit, auth_failure, ip_blocked, etc.).
- `api_ip_blocklist` ‚Äî Admin-managed IP blocklist.

### Routes

**Admin (site admin only):**
- `GET /admin/api` ‚Äî API monitoring dashboard
- `GET /admin/api/logs` ‚Äî Request logs JSON
- `GET /admin/api/security` ‚Äî Security events JSON
- `PUT /admin/api/security/:eventID/resolve` ‚Äî Resolve security event
- `POST /admin/api/ip-blocks` ‚Äî Block an IP
- `DELETE /admin/api/ip-blocks/:blockID` ‚Äî Unblock an IP
- `PUT /admin/api/keys/:keyID/toggle` ‚Äî Admin activate/deactivate key
- `DELETE /admin/api/keys/:keyID` ‚Äî Admin revoke key

**Campaign (campaign owner only):**
- `GET /campaigns/:id/api-keys` ‚Äî Key management page
- `POST /campaigns/:id/api-keys` ‚Äî Create new API key
- `PUT /campaigns/:id/api-keys/:keyID/toggle` ‚Äî Toggle key active/inactive
- `DELETE /campaigns/:id/api-keys/:keyID` ‚Äî Revoke key

**REST API v1 (API key auth):**
- `GET /api/v1/campaigns/:id` ‚Äî Campaign details (read)
- `GET /api/v1/campaigns/:id/entity-types` ‚Äî List entity types (read)
- `GET /api/v1/campaigns/:id/entity-types/:typeID` ‚Äî Get entity type (read)
- `GET /api/v1/campaigns/:id/entities` ‚Äî List entities (read, supports ?q=&type_id=&page=&per_page=)
- `GET /api/v1/campaigns/:id/entities/:entityID` ‚Äî Get entity (read)
- `POST /api/v1/campaigns/:id/entities` ‚Äî Create entity (write)
- `PUT /api/v1/campaigns/:id/entities/:entityID` ‚Äî Update entity (write)
- `PUT /api/v1/campaigns/:id/entities/:entityID/fields` ‚Äî Update fields only (write)
- `DELETE /api/v1/campaigns/:id/entities/:entityID` ‚Äî Delete entity (write)
- `POST /api/v1/campaigns/:id/sync` ‚Äî Bulk bidirectional sync (sync)
- `GET /api/v1/campaigns/:id/media` ‚Äî List media files (read, paginated)
- `GET /api/v1/campaigns/:id/media/:mediaID` ‚Äî Get media file metadata (read)
- `GET /api/v1/campaigns/:id/media/stats` ‚Äî Campaign storage stats (read)
- `POST /api/v1/campaigns/:id/media` ‚Äî Upload media file (write, multipart)
- `DELETE /api/v1/campaigns/:id/media/:mediaID` ‚Äî Delete media file (write)

## Security Design
- API keys are generated with `crypto/rand` (32 bytes + "chron_" prefix)
- Keys are bcrypt-hashed; plaintext shown once at creation, never stored
- Key prefix (first 8 chars) stored for lookup without exposing full key
- Per-key rate limits (configurable, default 60 req/min)
- Optional IP allowlist per key (CIDR notation)
- Optional key expiry dates
- Admin IP blocklist checked before processing any API request
- All requests logged with IP, user agent, status, latency
- Security events auto-logged for rate limits, auth failures, blocked IPs

## Middleware Stack
API v1 requests pass through this middleware chain:
1. `RequireAPIKey` ‚Äî Extract + validate Bearer token, check IP blocklist, log request.
2. `RateLimit` ‚Äî Fixed-window per-minute rate limiter with X-RateLimit headers.
3. `RequireCampaignMatch` ‚Äî Ensure key's campaign matches URL :id param.
4. `RequirePermission` ‚Äî Check key has the required permission (read/write/sync).

## Dependencies
- `campaigns` ‚Äî For campaign context middleware, role checks, and campaign info.
- `entities` ‚Äî For entity CRUD operations in API v1 endpoints.
- `auth` ‚Äî For authentication middleware.
- `apperror` ‚Äî For domain error types.
- `golang.org/x/crypto/bcrypt` ‚Äî For key hashing.

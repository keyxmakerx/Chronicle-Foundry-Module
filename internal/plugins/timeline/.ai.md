# Timeline Plugin

## Purpose

Interactive visual timelines for campaigns. Each campaign can have multiple
timelines that reference a calendar and display calendar events in a zoomable,
draggable visualization. Events are linked (not copied) from the calendar,
preserving a single source of truth. Entity groups organize events into
swim-lanes for the visualization.

## Tier

Plugin (Feature app with handler/service/repo/templates)

## Domain Concepts

- **Timeline:** A named visual timeline within a campaign. References a specific
  calendar for date context. Has its own color, icon, visibility, and zoom config.
- **Event Link:** A many-to-many join between timelines and calendar events. Supports
  per-link overrides for label, color, and visibility.
- **Entity Group:** A named group of entities used for swim-lane organization. Events
  linked to entities in a group appear in that group's lane.
- **Visibility Rules:** JSON-based per-user visibility overrides (allowed_users,
  denied_users) for fine-grained access control beyond everyone/dm_only.
- **Zoom Levels:** era, century, decade, year, month, day — each with distinct
  visual styles in the D3.js visualization (Sprint 3+).

## Design Decisions

- **ADR-018:** D3.js chosen over Leaflet.js for timeline visualization (Leaflet is
  geographic-only; D3 provides time scales, SVG rendering, zoom/pan, tree layouts)
- **Lens over calendar events:** Timeline events are links (not copies) to calendar
  events, preserving calendar as single source of truth
- **User-selected calendar:** Each timeline has a calendar_id FK allowing user to
  choose which calendar to reference (forward-compatible with multi-calendar)
- **Visibility rules as JSON:** Extends existing everyone/dm_only system with
  per-user allowed/denied lists, scoped to timeline plugin only

# Maps Plugin

## Purpose

Provides interactive map support for campaigns. Each campaign can have multiple
maps (world, region, city, dungeon) with background images displayed via
Leaflet.js. Maps contain positioned pin markers that optionally link to
entities. Phase 2 adds a drawing subsystem with vector drawings, tokens,
layers, and fog of war.

## Tier

**Plugin** -- feature app with handler, drawing handler, service, drawing
service, repository, drawing repository, models, and Templ templates.

## Files

| File | Role |
|------|------|
| `handler.go` | Map and marker CRUD handlers (web UI + JSON API) |
| `drawing_handler.go` | Drawing, token, layer, fog handlers (JSON API) |
| `model.go` | Map, Marker domain types and DTOs |
| `drawing.go` | Drawing, Token, Layer, FogRegion domain types and DTOs |
| `service.go` | MapService: map/marker business logic, XSS validation |
| `drawing_service.go` | DrawingService: drawing/token/layer/fog business logic |

| `drawing_repository.go` | DrawingRepository: MariaDB persistence for Phase 2 |
| `routes.go` | Route registration for web and drawing API |
| `templates/` | Templ components for map list and map viewer pages |

## Data Model

### Map
- `id` (UUID), `campaign_id`, `name`, `description`, `image_id` (media FK),
  `image_width`, `image_height`, `sort_order`, timestamps.

### Marker
- `id` (UUID), `map_id` (FK), `name`, `description`, `x`/`y` (percentage 0-100),
  `icon` (FontAwesome class), `color` (hex), `entity_id` (optional FK),
  `visibility` (`everyone` or `dm_only`), `created_by`, timestamps.
- Joined fields: `entity_name`, `entity_icon` from LEFT JOIN on entities.

### Drawing (Phase 2)
- Types: `freehand`, `rectangle`, `ellipse`, `polygon`, `text`.
- Percentage-based coordinates for resolution independence.
- Stroke/fill styling, rotation, layer assignment, DM-only visibility.
- Optional `foundry_id` for Foundry VTT sync integration.

### Token (Phase 2)
- Represents characters/NPCs/objects on the map.
- Position (`x`/`y`), size (`width`/`height`), rotation, scale.
- Combat support: `bar1_value/max`, `bar2_value/max` (HP bars).
- Visual effects: `aura_radius/color`, `light_radius/dim_radius/color`.
- Game mechanics: `vision_enabled/range`, `elevation`, `status_effects` (JSON).
- Optional `entity_id` link and `foundry_id` for sync.

### Layer (Phase 2)
- Types: `background`, `drawing`, `token`, `gm`, `fog`.
- `sort_order`, `is_visible`, `opacity`, `is_locked`.

### FogRegion (Phase 2)
- Polygon-based fog of war regions with `is_explored` flag.

## Web Endpoints

| Method | Path | Role | Description |
|--------|------|------|-------------|
| GET | `/campaigns/:id/maps` | Player | Map list page |
| GET | `/campaigns/:id/maps/:mid` | Player | Map viewer (Leaflet.js) |
| POST | `/campaigns/:id/maps/new` | Owner | Create map (form POST) |
| POST | `/campaigns/:id/maps` | Owner | Create map (JSON) |
| PUT | `/campaigns/:id/maps/:mid` | Owner | Update map metadata |
| DELETE | `/campaigns/:id/maps/:mid` | Owner | Delete map + markers (FK cascade) |
| POST | `/campaigns/:id/maps/:mid/markers` | Scribe | Create marker |
| PUT | `/campaigns/:id/maps/:mid/markers/:mkid` | Scribe | Update marker |
| DELETE | `/campaigns/:id/maps/:mid/markers/:mkid` | Owner | Delete marker |

## Drawing API Endpoints (Phase 2)

All under `/campaigns/:id/maps/:mid/`. Used by both the Chronicle web UI
and the Foundry VTT sync module.

| Resource | List | Create | Get | Update | Delete | Extra |
|----------|------|--------|-----|--------|--------|-------|
| Drawings | GET (Player) | POST (Scribe) | GET (Player) | PUT (Scribe) | DELETE (Owner) | -- |
| Tokens | GET (Player) | POST (Scribe) | GET (Player) | PUT (Scribe) | DELETE (Owner) | PATCH .../position (Scribe) |
| Layers | GET (Player) | POST (Owner) | GET (Player) | PUT (Owner) | DELETE (Owner) | -- |
| Fog | GET (Owner) | POST (Owner) | -- | -- | DELETE (Owner) | POST .../reset (Owner) |

## Key Functions

- `MapService.CreateMarker`: validates name, coordinates (0-100 range), icon
  pattern (`^fa-[a-z0-9-]+$`), and color pattern (`^#[0-9a-fA-F]{3,6}$`).
  Sets defaults: icon `fa-map-pin`, color `#3b82f6`, visibility `everyone`.
- `MapService.ListMarkers`: filters by role (role >= 3 sees dm_only markers).
- `MapService.SearchMaps`: name-based search returning results formatted for
  the quick search system (id, name, type_name, type_icon, url).
- `DrawingService.ListDrawings/ListTokens`: role-based visibility filtering.
- `DrawingService.UpdateTokenPosition`: optimized position-only update for
  real-time drag sync.

## Security

- **IDOR protection**: `requireMapInCampaign`, `requireMarkerInCampaign`, and
  `requireMapOwnership` verify ownership before every operation.
- **XSS prevention**: marker `icon` and `color` validated against regex patterns
  since they render into HTML attributes and CSS.
- **Addon gating**: all map routes require the `maps` addon via middleware.
- **Visibility**: dm_only markers/drawings/tokens hidden from non-Owner roles.

## Dependencies

- **Uses:** campaigns (access control), auth (authentication), addons (gating),
  media (map images), entities (marker linking)
- **Used by:** syncapi plugin (external map API), WebSocket real-time updates

## Notable Design Decisions

- Marker coordinates are percentages (0-100) not pixels, so maps work at any
  image resolution. Leaflet CRS.Simple converts to pixel space at render time.
- Phase 2 drawing subsystem uses separate `DrawingService` and `DrawingHandler`
  to keep the original map/marker code untouched.
- Token position has a dedicated PATCH endpoint optimized for real-time drag
  sync (smaller payload than full token update).
- Fog of war is Owner-only (GM controls what players see). Reset clears all
  fog regions for a map in one operation.
- `foundry_id` fields on drawings and tokens enable stable ID mapping for
  Foundry VTT sync without relying on Chronicle's internal UUIDs.
- Map deletion cascades to markers via FK constraint in MariaDB.

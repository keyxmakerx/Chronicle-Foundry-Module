# Extensions Package

## Purpose
Content extension system (Layer 1 of the three-layer extension architecture). Provides
manifest-based, declarative content packs with no code execution. Extensions contribute
calendar presets, entity type templates, entity packs, tag collections, relation types,
marker icon packs, themes, and reference data.

## Architecture
- **model.go** ‚Äî Core types: `Extension`, `CampaignExtension`, `Provenance`, `ExtensionData`
- **manifest.go** ‚Äî `ExtensionManifest` parsing/validation (semver, slug, file path safety)
- **security.go** ‚Äî Zip extraction with path traversal prevention, file type allowlist,
  CSS/SVG sanitization, size limits
- **repository.go** ‚Äî MariaDB CRUD for 4 tables (extensions, campaign_extensions,
  extension_provenance, extension_data)
- **service.go** ‚Äî Business logic: install, uninstall, update, rescan, per-campaign
  enable/disable, provenance tracking, extension data storage
- **handler.go** ‚Äî HTTP handlers for admin (install/uninstall/update/rescan/list) and
  campaign (enable/disable/list/preview) operations, plus static asset serving
- **routes.go** ‚Äî Route registration for admin, campaign, and asset routes
- **admin_extensions.templ** ‚Äî Admin UI for extension management

## Key Design Decisions
- **Hybrid storage**: Extensions write to existing tables (entity_types, tags, calendars)
  with provenance tracked in `extension_provenance`
- **Per-campaign activation**: Site admin installs globally; campaign owners enable/disable
- **Security-first**: File type allowlist (.json, .css, .svg, .png, .webp, .jpg, .jpeg,
  .txt, .md), path traversal prevention, SVG script/event handler blocking, CSS @import/
  url()/expression() blocking
- **Size limits**: 50MB zip, 100MB extracted, 1000 files, 20MB per file, 500KB per CSS

## Tables (migration 000055)
- `extensions` ‚Äî Installed extensions (manifest, version, status)
- `campaign_extensions` ‚Äî Per-campaign enable/disable state
- `extension_provenance` ‚Äî Tracks which records were created by which extension
- `extension_data` ‚Äî Key-value storage for extension state per campaign

## API Routes
### Admin (`/admin/extensions`)
- `GET /` ‚Äî List all extensions
- `POST /install` ‚Äî Install from zip upload
- `PUT /:ext_id` ‚Äî Update extension from zip
- `DELETE /:ext_id` ‚Äî Uninstall extension
- `POST /rescan` ‚Äî Discover extensions from filesystem

### Campaign (`/campaigns/:id/extensions`)
- `GET /` ‚Äî List extensions with campaign enable/disable status
- `GET /:ext_id/preview` ‚Äî Preview what extension contributes
- `POST /:ext_id/enable` ‚Äî Enable for campaign
- `POST /:ext_id/disable` ‚Äî Disable for campaign

### Assets (`/extensions/assets`)
- `GET /:ext_id/*` ‚Äî Serve extension static assets (path traversal protected)

## Dependencies
- `internal/apperror` ‚Äî Domain error types
- `internal/templates/layouts` ‚Äî Page layout components (via templ)
- No dependency on other plugins ‚Äî extensions are a standalone subsystem

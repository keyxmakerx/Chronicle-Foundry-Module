# Tags Widget

## Purpose

Campaign-scoped entity tagging system. Tags are colored labels that can be
attached to entities for categorization and filtering. The frontend tag picker
widget displays tags as colored chips with a dropdown for search, toggle, and
creation. Supports DM-only tags visible only to Scribes and Owners.

## Tier

**Widget** -- reusable UI block with Go backend and JS frontend. Auto-mounted
by `boot.js` on elements with `data-widget="tag-picker"`.

## Files

| File | Role |
|------|------|
| `static/js/widgets/tag_picker.js` | Client-side tag picker widget |
| `internal/widgets/tags/handler.go` | Go handler for tag CRUD and entity tag management |
| `internal/widgets/tags/service.go` | TagService: validation, slug generation, diff-based assignment |
| `internal/widgets/tags/model.go` | Tag struct, CreateTagRequest, SetEntityTagsRequest DTOs |
| `internal/widgets/tags/repository.go` | MariaDB persistence for tags and entity_tags tables |
| `internal/widgets/tags/routes.go` | Campaign-scoped routes with role-based access |
| `internal/widgets/tags/templates/` | Templ mount-point |

## Dependencies

- **Uses:** campaigns (CampaignContext, role checks), auth (user ID for audit)
- **Uses:** audit plugin (fire-and-forget logging of tag create/delete)
- **Used by:** Entity profile page (entities plugin), entity list cards (batch fetch)

## Mount Syntax

```html
<div data-widget="tag-picker"
     data-tags-endpoint="/campaigns/{cid}/tags"
     data-entity-tags-endpoint="/campaigns/{cid}/entities/{eid}/tags"
     data-editable="true"
     data-csrf-token="{token}">
</div>
```

## Configuration

| Attribute | Required | Description |
|-----------|----------|-------------|
| `data-tags-endpoint` | Yes | Campaign tags endpoint (GET list, POST create) |
| `data-entity-tags-endpoint` | Yes | Entity tags endpoint (GET current, PUT replace) |
| `data-editable` | No | Enable add/remove UI (Scribe+ only) |
| `data-csrf-token` | No | CSRF token for mutations |

## How It Works

### Frontend Behavior

1. **Init**: fetches campaign tags and entity tags in parallel via `Promise.all`.
2. **Display**: renders entity tags as colored chips (`background: color+22`
   alpha overlay, text in tag color). DM-only tags show eye-slash badge.
3. **Add button**: opens dropdown with search input. Filters tags by name as
   user types. Clicking a tag toggles it on/off.
4. **Create**: when search term does not match any existing tag, shows "Create"
   option with a DM-only checkbox. Assigns a random color from a 10-color
   palette. Creates tag then immediately attaches it to the entity.
5. **Remove**: clicking the X on a chip removes that tag from the entity.
6. **Close**: dropdown closes when clicking outside (document-level listener).

### Tag Colors

Random color from: `#6366f1`, `#8b5cf6`, `#ec4899`, `#f43f5e`, `#f97316`,
`#eab308`, `#22c55e`, `#14b8a6`, `#06b6d4`, `#3b82f6`.

### Scoped Styles

Injects a `<style id="tag-picker-styles">` element once on first mount.
Supports dark mode via `.dark` class selectors.

## API Endpoints

| Method | Path | Role | Description |
|--------|------|------|-------------|
| GET | `/campaigns/:id/tags` | Player | List campaign tags (Players see public only) |
| POST | `/campaigns/:id/tags` | Scribe | Create tag `{name, color, dmOnly}` |
| PUT | `/campaigns/:id/tags/:tagId` | Scribe | Update tag `{name, color, dmOnly}` |
| DELETE | `/campaigns/:id/tags/:tagId` | Owner | Delete tag |
| GET | `/campaigns/:id/entities/:eid/tags` | Player | Get entity's tags |
| PUT | `/campaigns/:id/entities/:eid/tags` | Scribe | Replace entity tags `{tagIds: [...]}` |

## Data Flow

1. Widget mounts, fetches `GET /tags` and `GET /entities/:eid/tags` in parallel.
2. User clicks a tag in dropdown -> widget sends `PUT /entities/:eid/tags`
   with the complete set of tag IDs (idempotent replacement).
3. Server diffs the current vs new tag set, applies minimal DB changes.
4. Server returns the updated tag list; widget re-renders chips.
5. Tag creation: `POST /tags` -> server returns new tag -> widget calls
   `PUT /entities/:eid/tags` to attach it.

## Key Backend Functions

| Function | Description |
|----------|-------------|
| `Handler.ListTags` | Returns tags filtered by role (Players miss dm_only) |
| `Handler.CreateTag` | Creates tag, logs audit event |
| `Handler.SetEntityTags` | Idempotent tag replacement, returns updated list |
| `Handler.GetEntityTags` | Returns entity's tags, role-filtered |
| `Handler.logAudit` | Fire-and-forget audit entry for tag mutations |

## Security

- **IDOR protection**: `UpdateTag` and `DeleteTag` verify tag belongs to the
  campaign before operating.
- **DM-only filtering**: `canSeeDmOnly` checks if role >= Scribe. Players
  cannot see dm_only tags in any endpoint response.
- **Audit trail**: tag creation and deletion logged with campaign ID, user ID,
  and tag name.

# Editor Widget

## Purpose

TipTap-based rich text editor for entity content. Mounts to DOM elements,
loads content from an API endpoint, provides WYSIWYG editing with @mentions
and entity linking, and saves content back via API. Starts in read-only view
mode by default; users click "Edit" to enter edit mode with full toolbar.

## Tier

**Widget** -- reusable UI block, auto-mounted by `boot.js` on elements with
`data-widget="editor"`.

## Files

| File | Role |
|------|------|
| `static/js/widgets/editor.js` | Client-side widget (TipTap init, toolbar, save/load) |
| `static/js/widgets/editor_secret.js` | TipTap secret mark extension (GM-only inline text) |
| `static/vendor/tiptap-bundle.min.js` | Vendored TipTap build |
| `internal/widgets/editor/templates/` | Templ mount-point (renders `data-widget="editor"`) |

## Dependencies

- **Uses:** TipTap bundle (`tiptap-bundle.min.js` must be loaded first)
- **Uses:** Entity entry API (`GET/PUT {endpoint}`) for load/save
- **Uses:** Mentions widget (`editor_mention.js`) via `Chronicle.MentionExtension`
- **Uses:** SecretMark extension (if loaded) for GM-only inline secrets
- **Used by:** Entity profile page (entities plugin)

## Mount Syntax

```html
<div data-widget="editor"
     data-endpoint="/campaigns/{cid}/entities/{eid}/entry"
     data-campaign-id="{cid}"
     data-editable="true"
     data-autosave="30"
     data-csrf-token="{token}">
</div>
```

## Configuration

| Attribute | Required | Description |
|-----------|----------|-------------|
| `data-endpoint` | Yes | API URL for GET (load) and PUT (save) |
| `data-campaign-id` | No | Campaign ID, enables @mention entity search |
| `data-editable` | No | `true` to allow editing (default: `false`) |
| `data-autosave` | No | Autosave interval in seconds (default: 30, 0 to disable) |
| `data-csrf-token` | No | CSRF token for PUT requests |

## How It Works

### TipTap Extensions

| Extension | Configuration |
|-----------|--------------|
| StarterKit | Headings H1-H3, link and underline excluded (replaced below) |
| Placeholder | "Begin writing your entry..." |
| Link / MentionLink | `openOnClick: true`. MentionLink preserves `data-mention-id` through JSON round-trips |
| Underline | Standard underline mark |
| Table, TableRow, TableCell, TableHeader | Resizable tables with `chronicle-table` class |
| SecretMark | GM-only inline secrets (if loaded) |

### View/Edit Mode Toggle

1. **View mode** (default): toolbar hidden, editor non-editable, links clickable,
   no autosave timer. Header shows "Edit" button.
2. **Edit mode**: toolbar visible, editor editable, autosave timer running,
   status bar shows save state. Header shows "Done" button. CSS class
   `chronicle-editor--editing` added for visual cues.

### Toolbar Groups

1. **Text formatting**: Bold, Italic, Underline, Strikethrough, GM Secret
2. **Block formatting**: H1, H2, H3
3. **Lists**: Bullet list, Numbered list
4. **Misc**: Blockquote, Code block, Horizontal rule
5. **History**: Undo, Redo
6. **Insert menu** (+): Link Entity (@mention), Insert Link, Horizontal Rule,
   Callout Block, Code Block, Insert Table (3x3 with header row)
7. **Save button**: highlights when unsaved changes exist

Active toolbar states update on `selectionUpdate` and `transaction` events.

### Content Storage

- Content stored as ProseMirror JSON in entity `entry` column.
- Pre-rendered HTML sent alongside as `entry_html` for display performance.
- `GET {endpoint}` returns `{ entry: "JSON string" }`.
- `PUT {endpoint}` sends `{ entry: "JSON string", entry_html: "HTML string" }`.

### Autosave

Timer fires every N seconds (default 30) in edit mode. If `dirty` flag is set,
PUTs to the endpoint. Manual save via toolbar Save button or Ctrl+S. Exiting
edit mode saves immediately.

### @Mention Integration

When `data-campaign-id` is set and `Chronicle.MentionExtension` is loaded:
- Extension initialized with `campaignId`, hooked into editor `update` events.
- `editorProps.handleKeyDown` intercepts arrow/Enter/Escape for popup navigation.
- Insert menu "Link Entity" action types `@` at cursor to trigger the popup.

### Dirty State Tracking

Calls `Chronicle.markDirty('editor')` on content changes and
`Chronicle.markClean('editor')` after save. Integrates with the global
unsaved-changes warning system.

## Key Functions

| Function | Description |
|----------|-------------|
| `init(el, config)` | Creates editor, toolbar, loads content from API |
| `destroy(el)` | Saves unsaved changes, clears timers, removes listeners |
| `enterEditMode(state)` | Shows toolbar, enables editing, starts autosave |
| `exitEditMode(state)` | Saves, hides toolbar, makes read-only |
| `createToolbar()` | Builds toolbar DOM with button groups and insert menu |
| `createInsertMenu()` | Builds + dropdown with discoverable features |
| `executeCommand(editor, cmd)` | Dispatches toolbar button commands to TipTap |
| `executeInsert(state, action)` | Handles insert menu actions |
| `loadContent(state)` | GET from endpoint, parse JSON, set editor content |
| `saveContent(state)` | PUT JSON + HTML to endpoint with CSRF token |

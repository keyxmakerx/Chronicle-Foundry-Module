# Draw Steel Module

## Purpose

Provides Draw Steel RPG reference content by MCDM Productions. Serves
ability descriptions, creature stat blocks, ancestry features, and kit
definitions as tooltips (on @mention hover) and dedicated reference pages.
Includes full character preset with Foundry VTT field mappings for
bidirectional actor sync.

## Tier

Module (Game System -- optional, enabled per campaign)

## Domain Concepts

- **CC-BY Content:** Creative Commons Attribution content from MCDM's SRD.
  We can include CC-BY-4.0 licensed content with attribution.
- **Tooltip:** A small HTML fragment shown on hover when a user @mentions a
  game reference (e.g., `@ability:tactical-strike` shows keywords, damage).
- **Reference Page:** A full page showing complete details for an ability,
  creature, ancestry, etc.
- **Characteristics:** Draw Steel uses 5 characteristics (Might, Agility,
  Reason, Intuition, Presence) instead of D&D's 6 ability scores.
- **Stamina:** Replaces HP. Characters are "winded" at half max.
- **Recoveries:** Spend to regain 1/3 max stamina. Limited resource.
- **Heroic Resource:** Class-specific power fuel (each class names it differently).
- **Kit:** Equipment loadout that provides armor, speed, stability bonuses.
- **EV (Encounter Value):** Monster difficulty rating (replaces CR).

## Files

| File | Purpose |

## Dependencies

- **Uses:** Nothing (read-only, self-contained)
- **Used by:** widgets/mentions (tooltip API), entity pages (reference links),
  Foundry module (generic adapter reads character-fields API)

## Business Rules

- **Read-only.** This module NEVER writes to the database.
- Only CC-BY-4.0 licensed content with MCDM attribution.
- Tooltip API returns HTML fragments for hover previews.
- Data files are loaded once at startup and cached in memory.
- Character preset uses `foundry_actor_type: "hero"` (not "character") to
  match the Draw Steel Foundry system's actor type.
- Foundry system ID is `"draw-steel"` (hyphenated).

## Foundry Integration

The character preset includes `foundry_path` annotations on all syncable
fields. The generic adapter reads these via the `/systems/drawsteel/character-fields`
API endpoint and auto-generates bidirectional field mappings. No hand-written
adapter is needed.

### Key Foundry Paths

| Chronicle Field | Foundry Path | Writable |
|----------------|-------------|----------|
| might | system.characteristics.might.value | yes |
| agility | system.characteristics.agility.value | yes |
| reason | system.characteristics.reason.value | yes |
| intuition | system.characteristics.intuition.value | yes |
| presence | system.characteristics.presence.value | yes |
| stamina_current | system.stamina.value | yes |
| stamina_max | system.stamina.max | no (derived) |
| heroic_resource | system.hero.primary.value | yes |
| surges | system.hero.surges | yes |
| victories | system.hero.victories | yes |
| speed | system.movement.value | no (derived) |
| stability | system.combat.stability | no (derived) |
